# Career Path Recommendation System

A full-stack AI-powered career guidance platform built with **Next.js 16**, **TypeScript**, and **MongoDB**. The system helps users discover suitable career paths, generate personalized learning roadmaps, compare careers side-by-side, and interact with an intelligent career advisor chatbot — all backed by a multi-provider AI engine with automatic fallback and rate-limit protection.

## Overview

The Career Path Recommendation System is a comprehensive web application that enables users to:

- Explore a curated library of careers across tech, management, and non-tech domains
- Get personalized career recommendations based on their education, skills, and interests
- Generate AI-powered, stage-aware learning roadmaps tailored to their current life stage (school, college, working professional, etc.)
- Chat with an AI career advisor for real-time guidance
- Compare multiple careers side-by-side on salary, difficulty, growth, and skills
- Save careers and roadmaps to their personal profile
- Submit feedback through a contact form

---

## Features

### Core Features

- **Career Discovery** — Browse careers filtered by domain, difficulty, growth rate, and skills
- **AI Career Recommendations** — Algorithmic matching based on education level, interests, and skills with weighted scoring
- **AI Roadmap Generator** — Fully personalized, multi-stage learning roadmaps generated by Gemini/Groq/OpenRouter with decision breakpoints, free/paid resource tracks, and alternate paths
- **AI Career Advisor Chatbot** — Streaming chat interface powered by a multi-provider AI engine with topic filtering (career-only context)
- **Career Comparison Tool** — Side-by-side comparison of up to multiple careers on key metrics
- **User Profiles** — Save favorite careers and roadmaps, update personal details (education, skills, interests)
- **Feedback / Contact** — Persistent feedback storage in MongoDB

### Admin Features

- Full CRUD management of careers via the Admin Panel
- User management (view/delete users)
- Protected admin routes with token-based setup

### Technical Features

- **Multi-Provider AI with Fallback Chain** — Gemini → Groq → Bytez → OpenRouter, each with independent rate limiting and circuit breakers
- **In-Memory Response Cache** — 2-minute TTL, 250-entry LRU-style cache for chat responses
- **Demo Mode** — Fully functional in-memory store when MongoDB is unavailable
- **JWT Authentication** — Custom HS256 JWT implementation with httpOnly cookies
- **Route Protection** — Middleware-based authentication with public/private route classification
- **Responsive UI** — Mobile-first design using Tailwind CSS v4 and shadcn/ui component library

## AI Engine & Multi-Provider Architecture

The platform uses a cascading multi-provider AI system with built-in resilience. Requests flow through a fallback chain until a successful response is obtained.

### Provider Fallback Chain

```
Request
  │
  ▼
[1] Google Gemini  ──(fail/429/timeout)──►
[2] Groq           ──(fail/429/timeout)──►
[3] Bytez          ──(fail/429/timeout)──►
[4] OpenRouter     ──(fail)──► Error Response
```

### Guardrails (lib/ai-guardrails.ts)

Each provider is independently managed by:

- **Rate Limiter** — Sliding window counter tracking requests per minute per provider
- **Circuit Breaker** — If a provider fails (429 / 503) beyond a failure threshold, it enters a cooldown period and is skipped until the cooldown expires
- **Provider Queue** — Sequential request queue per provider to prevent concurrent overload
- **Timeout** — Per-provider request timeout (Gemini: 15s, Groq: 12s, OpenRouter: 15s)

### Chat Response Cache

- TTL: 2 minutes
- Max entries: 250
- Keyed on message content hash
- Prevents redundant AI calls for repeated questions

### Topic Guardrails (Chat)

The chat API filters user messages against a career-related keyword list. Off-topic messages receive a polite redirect without consuming AI quota.

### Roadmap Generation

Roadmaps are generated with the user's profile context:

- Date of birth / Age
- Current education stage (`school-10`, `intermediate-11-12`, `diploma`, `ug`, `working`, `other`)
- Current qualification
- Current role (if working)
- Skills list

The AI response includes per-step structured data: title, description, duration, free resources, paid resources, decision breakpoints, alternate paths, and a full pathway summary.

## Authentication & Security

- **Password Hashing** — bcryptjs with salt rounds
- **JWT** — Custom HS256 implementation using Node.js built-in `crypto` (HMAC-SHA256), no external JWT library
- **Token Storage** — `httpOnly`, `sameSite: lax`, `secure` (production), 7-day expiry cookie
- **Timing-Safe Comparison** — `timingSafeEqual` used for signature verification to prevent timing attacks
- **Route Protection** — `middleware.ts` intercepts all non-public routes; unauthenticated API calls return `401`, unauthenticated page visits redirect to `/login?redirect=<path>`
- **Admin Protection** — Admin routes additionally validate `isAdmin` flag from JWT payload

## Environment Variables

Create a `.env.local` file in the project root with the following variables:

```env
# ── Database ────────────────────────────────────────────────
MONGODB_URI=mongodb+srv://<user>:<password>@cluster.mongodb.net/<dbname>

# ── Authentication ──────────────────────────────────────────
JWT_SECRET=your_very_long_random_secret_key_here

# ── Application URLs ─────────────────────────────────────────
NEXT_PUBLIC_APP_URL=http://localhost:3000
NEXT_PUBLIC_API_URL=http://localhost:3000

# ── Google Gemini (Primary AI Provider) ─────────────────────
GEMINI_API_KEY=your_gemini_api_key
GEMINI_API_KEY_FALLBACK=your_second_gemini_key         # optional
GEMINI_API_KEY_FALLBACK_2=your_third_gemini_key        # optional
GEMINI_API_KEY_FALLBACK_3=your_fourth_gemini_key       # optional
GEMINI_MODEL=gemini-2.0-flash                          # optional override

# ── Groq (Secondary AI Provider) ────────────────────────────
GROQ_API_KEY=your_groq_api_key
GROQ_MODEL=llama-3.1-8b-instant                        # optional override

# ── Bytez (Tertiary AI Provider) ─────────────────────────────
BYTEZ_API_KEY=your_bytez_api_key
BYTEZ_MODEL=openai/gpt-4.1-mini                        # optional override

# ── OpenRouter (Final Fallback AI Provider) ──────────────────
OPENROUTER_API_KEY=your_openrouter_api_key
OPENROUTER_MODEL=google/gemma-3-27b-it:free            # must end with :free
OPENROUTER_FREE_MODELS=google/gemma-3-27b-it:free      # comma-separated list
OPENROUTER_REFERER=http://localhost:3000
OPENROUTER_TITLE=PathFinder

# ── Admin ────────────────────────────────────────────────────
ADMIN_SETUP_TOKEN=your_secret_admin_setup_token
```

> **Note:** At minimum, `JWT_SECRET` and at least one AI provider key (`GEMINI_API_KEY` recommended) are required. `MONGODB_URI` is optional — the app runs in demo mode with an in-memory store if omitted.

## Getting Started

### Prerequisites

- Node.js 18+
- pnpm (recommended) or npm
- MongoDB Atlas account or local MongoDB instance (optional for demo mode)

### Installation

```bash
# Clone the repository
git clone https://github.com/your-username/career-path-recommendation-system.git
cd career-path-recommendation-system

# Install dependencies
pnpm install

# Set up environment variables
cp .env.example .env.local
# Edit .env.local with your values

# Run development server
pnpm dev
```

Open [http://localhost:3000](http://localhost:3000) in your browser.

### Build for Production

```bash
pnpm build
pnpm start
```

### Alternative Dev with Webpack

```bash
pnpm dev:webpack
```

## Admin Panel

### Initial Admin Setup

After deployment, initialize the admin user by calling:

```
POST /api/admin/setup
Authorization: Bearer <ADMIN_SETUP_TOKEN>
```

This creates the default admin account. The `ADMIN_SETUP_TOKEN` must match the value in your `.env.local`.

### Admin Capabilities

- **Career Management** — Create, update, and delete careers from the UI
- **User Management** — View all registered users, delete accounts
- **Dashboard** — Overview of platform stats

Access the admin panel at `/admin` after logging in with an admin account.

## Demo Mode (No Database)

If `MONGODB_URI` is not set, the application automatically falls back to an **in-memory store** defined in `lib/store.ts`. This store is pre-seeded with:

- 1 admin user (`admin@career.com`)
- 8 sample careers (AI/ML Engineer, Full Stack Developer, Data Scientist, Cloud Architect, Cybersecurity Analyst, Product Manager, UX/UI Designer, DevOps Engineer)

This allows the app to be fully explored and demonstrated without any database setup. Note that data does not persist between server restarts in demo mode.
